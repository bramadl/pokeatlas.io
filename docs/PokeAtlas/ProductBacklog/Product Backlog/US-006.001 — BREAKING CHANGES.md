## 2. Arsitektur: Views + Filters + Brush Toolbar

### Tiga konsep yang harus tetap terpisah

**Filter** — mengubah *data apa* yang muncul di grid. Menjawab pertanyaan: "Pokémon mana yang ingin aku lihat?" Contoh: hanya Kanto, hanya yang belum di-track, hanya tipe Fire.

**View** — mengubah *konteks tracking* yang aktif. Menjawab pertanyaan: "Aku lagi track dimensi apa sekarang?" Satu View = satu primary lens. Badge dan status di card menyesuaikan View yang aktif.

**Brush Toolbar** — tetap ada, mengubah *state apa* yang di-apply saat tap. Memberikan fleksibilitas tracking dalam satu View.

### Keputusan: Opsi B (View-centric) dengan Brush Toolbar dipertahankan

View yang aktif menentukan *konteks tampilan*, Brush yang aktif menentukan *apa yang di-track saat tap*. Keduanya bisa berbeda — misalnya, berada di Hundo View tapi pakai Shiny Brush untuk nge-track shiny yang baru dapat.

Alasan mempertahankan Brush Toolbar:

- Shiny Hundo harus bisa di-track dari View manapun (Shiny View atau Hundo View)
- Multi-state tracking dalam satu sesi tetap dimungkinkan
- Constraint brush per View (lihat bagian 2.3) menggantikan kebutuhan menghapus toolbar

### View navigator — 7 views, bukan 24

Views bukan permutation engine. Views mencerminkan *intent / primary lens* tracking, bukan semua kombinasi state yang mungkin. Kombinasi seperti "Shiny Hundo" tetap bisa di-track dari View manapun via Brush — hasilnya akan muncul di kedua Views secara bersamaan.

```txt
Base · Shiny · Hundo · Nundo · Shadow · Purified · Lucky
```

Contoh perilaku intersection:

- Pokemon Shiny Hundo → muncul di Shiny View (✓) DAN Hundo View (✓)
- Pokemon Shiny saja → muncul di Shiny View (✓), tidak di Hundo View (–)

### Brush constraints per View

Brush Toolbar harus memiliki constraint kontekstual — brush yang tidak logis secara domain di-disable otomatis tergantung View aktif.

| View aktif    | Brush yang disabled    | Alasan                                            |
|---------------|------------------------|---------------------------------------------------|
| Shadow View   | Purified               | Pokemon Shadow tidak bisa sekaligus Purified      |
| Shadow View   | Lucky                  | Pokemon Shadow tidak dapat di-trade               |
| Purified View | Shadow                 | Sudah dimurnikan, tidak bisa Shadow               |
| Purified View | Nundo                  | Sudah dimurnikan, dapat +2 on all stats           |
| Base View     | —                      | Semua brush aktif                                 |
| Shiny View    | —                      | Semua brush aktif (shiny bisa dikombinasi apapun) |
| Hundo View    | Nundo                  | Hundo tidak bisa menjadi Nundo                    |
| Nundo View    | Hundo                  | Nundo tidak bisa menjadi Hundi                    |
| Nundo View    | Purified               | Purified pokemon dari Nundo menjadi 2/2/2         |
| Lucky View    | Shadow                 | Semua brush aktif kecuali Shadow                  |

> **Note untuk implementasi:** Constraint ini menyentuh domain logic, bukan hanya UI. Perlu ada validasi di layer yang sama dengan `TrackedStates` / `trackable-state.ts`.

### Monetization opportunity (opsional, FYI)

Pembatasan Views per tier adalah monetization yang legit:

- **Free:** Base, Shiny, Hundo
- **Premium:** Semua 7 Views + filter lanjutan

Ini belum keputusan final, dicatat sebagai opsi.

## 3. Filter Panel — Struktur Baru

### Status filter (single-select, context-aware)

Label berubah tergantung View aktif:

| View        | Pilihan                       |
|-------------|-------------------------------|
| Base View   | All · Tracked · Missing       |
| Hundo View  | All · Has hundo · No hundo    |
| Shiny View  | All · Has shiny · No shiny    |
| Shadow View | All · Has shadow · No shadow  |
| *(dst)*     | All · Has [view] · No [view]  |

### Form filter (single-select, seperti sekarang)

```txt
(All) · Mega · Primal · Alternate
```

- **Temporary Evolution Forms** — Mega Evolution (Charizard X/Y, Gengar, dll) termasuk Primal Reversion (Kyogre, Groudon) — mekanisme berbeda dari Mega tapi diperlakukan serupa di GO
- **Alternate Forms** — satu bucket untuk semua alternate form yang catchable langsung: Origin Dialga/Palkia, Zacian Crowned, Landorus Therian, dll. User tidak perlu tau nama teknisnya.

Form yang **tidak di-expose** di filter:

- Battle-only forms (Aegislash Blade/Shield, Wishiwashi School, Mimikyu Busted, Morpeku Full Belly/Hungry) — gak ada sebagai trackable entry di GO.
- Dynamax/Gigantamax — belum masuk GO sebagai trackable form; tambahkan nanti kalau relevan.
<!-- - Eternamax Eternatus — sama, battle-only. *NOTE*: Masih dipertimbangkan. -->

### Variant Settings (multi-select checkbox, baru)

Terpisah dari Form karena secara semantik berbeda — ini bukan "form lain", tapi "variant dari species yang sama".

| Variant             | Default | Contoh                                                                            |
|---------------------|---------|-----------------------------------------------------------------------------------|
| Regional variants   | ✓ on    | Alolan, Hisuian, Galarian, Paldean                                                |
| Appearance variants | ✓ on    | Flabébé (Red/Yellow/Blue/Orange/White), Basculin (Red/Blue Stripe), Furfrou trims |
| Gender variants     | ✓ on    | Female Frillish, Female Meowstic, Female Unfezant                                 |
| Costume variants    | ✗ off   | Bulbasaur (Fall 2019), Pikachu (PhD Hat), dll                                     |

Costume default off karena kebanyakan trainer tidak aktif hunting costume variants.

### Region filter (multi-select, baru)

Implementasi via dex number range — tidak perlu migration, cukup `WHERE dexNumber BETWEEN x AND y`.

| Region | Dex range |
|--------|-----------|
| Kanto  | 1–151     |
| Johto  | 152–251   |
| Hoenn  | 252–386   |
| Sinnoh | 387–493   |
| Unova  | 494–649   |
| Kalos  | 650–721   |
| Alola  | 722–809   |
| Galar  | 810–905   |
| Hisui  | 906–905*  |
| Paldea | 906–1025  |

> *Pokémon Hisui bukan region terpisah dalam numbering — mereka adalah form dari Pokemon existing. Filter Hisui kemungkinan lebih tepat masuk Variant: Regional daripada Region filter. TBD.

### Types filter (multi-select, sudah ada)

Tidak berubah dari implementasi sebelumnya.

## 4. Klasifikasi Form Pokemon GO — Referensi

Panduan untuk edge cases saat menambahkan data baru:

**Tanya**: "Apakah ini beda Pokédex entry yang bisa di-track secara terpisah di GO?"

- Ya, beda entry, beda mekanik (Mega/Primal/Alternate) → **Form**
- Ya, beda entry, sama species (Alolan Vulpix vs Vulpix) → **Variant: Regional**
- Ya, beda entry, beda warna/stripe (Flabébé Red vs Yellow) → **Variant: Appearance**
- Ya, beda entry, beda gender sprite (Female Frillish) → **Variant: Gender**
- Ya, beda entry, limited event (Pikachu PhD Hat) → **Variant: Costume**
- Tidak, terjadi otomatis saat battle (Aegislash Blade) → **Exclude dari tracking**
