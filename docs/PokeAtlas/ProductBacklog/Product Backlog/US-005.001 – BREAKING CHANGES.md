# Pokedex Screen — Technical Redesign Spec

## Context & Decisions

### Two Orthogonal Concepts (Previously Conflated)

| Concept          | What it is                                              | Where it lives         |
|------------------|---------------------------------------------------------|------------------------|
| **Dex**          | *Game-aware* scope — "which collection am I browsing"   | Toolbar (Dex Switcher) |

## [1] Dex Switcher

### Concept

A "Dex" defines **which Pokémon are visible and how they are grouped**.
It is not a filter — it is a *lens* that changes the scope of the entire list.

### Dex Definitions

| Dex          | Query logic                                                           | Grouping                        |
|--------------|-----------------------------------------------------------------------|---------------------------------|
| **National** | All base species + all forms per Variant Controls                     | By generation (Gen 1–9 headers) |
| **Kanto**    | `dexNumber 1–151`, no regional forms                                  | Flat list                       |
| **Johto**    | `dexNumber 152–251`, no regional forms                                | Flat list                       |
| **Hoenn**    | `dexNumber 252–386`, no regional forms                                | Flat list                       |
| **Sinnoh**   | `dexNumber 387–493`, no regional forms                                | Flat list                       |
| **Unova**    | `dexNumber 494–649`, no regional forms                                | Flat list                       |
| **Kalos**    | `dexNumber 650–721`, no regional forms                                | Flat list                       |
| **Alola**    | `dexNumber 722–809` + `formCategory=REGIONAL_FORM AND origin=ALOLA`   | Flat list                       |
| **Galar**    | `dexNumber 810–905` + `formCategory=REGIONAL_FORM AND origin=GALAR`   | Flat list                       |
| **Hisui**    | `formCategory=REGIONAL_FORM AND origin=HISUI` only                    | Flat list                       |
| **Paldea**   | `dexNumber 906–1025` + `formCategory=REGIONAL_FORM AND origin=PALDEA` | Flat list                       |

**Note on Hisui:**
> No dex number range exists. Hisuian forms share dex numbers
> with their base species (e.g. Hisuian Growlithe = #58). Query is purely by
> `formCategory + origin`.

**Note on Meltan/Melmetal (#808–809):**
> Kept under Alola range for ordering
> consistency. Labeled "Unknown" origin in GO — acceptable tradeoff.

### URL Param

```txt
?dex=national   (default, omitted from URL)
?dex=kanto
?dex=hisui
...
```

Single value, not an array. Replaces the current `regions` array param for this concept.

### UI Behavior

```txt
[←]  Kanto Dex  [→]
```

- Prev/Next cycles through the ordered dex list
- Clicking the label opens a dropdown to jump directly
- Defaults to National Dex (no URL param needed)

## [2] Filter Panel Changes

### Remove: Form Filter

`formCategory` filter is removed from the panel entirely. It was confusing:

- "Regional" overlapped with Region/Dex concept
- "Alternate" is better served as a Variant Control toggle
- "Base" was rarely useful as a standalone filter

### Add: Classification Filter

Replaces Form filter. Uses existing `PokemonClassification` enum from DB.

```ts
// Already in schema:
enum PokemonClassification {
  LEGENDARY
  MYTHIC
  ULTRA_BEAST
}
```

UI: three toggle buttons — Legendary, Mythic, Ultra Beast. Multi-select.
Default: all included (no filter applied = show everything).

### Unchanged

- Status filter (Missing / Tracked)
- Types filter (up to 2 types)
- Regions filter → **removed**, replaced by Dex Switcher

### Final Filter Panel

```txt
STATUS          [Missing] [Tracked]
─────────────────────────────────
CLASSIFICATION  [Legendary] [Mythic] [Ultra Beast]  
─────────────────────────────────
TYPES           [bug][dark][dragon]...(18 types, max 2)
```

## [3] Variant Controls Changes

### Add: Alternate Forms toggle

```ts
// New entry in VARIANT_DEFINITIONS:
{
  key: "alternateForm",
  urlValue: "alternate_form",
  storageKey: "pokedex.filter.variant.alternateForm",
}
```

**Default: OFF** — keeps the list clean. User opts in to see Arceus-Flying,
Unown-A through Z, Spinda variants, etc.

**Query change in BE adapter:**

```ts
// Currently only costume/gender/temporaryEvolution are controlled.
// Add:
const { alternateForm = false } = filters?.variants ?? {};
if (!alternateForm) andConditions.push({ formCategory: { not: "ALTERNATE_FORM" } });
```

> **Important:** Costume Pokémon have `formCategory = ALTERNATE_FORM` + `isCostume = true`.
> The existing `isCostume` filter already handles costumes separately.
> So `alternateForm = false` should be:
> `formCategory != ALTERNATE_FORM OR isCostume = true`
> to avoid hiding costumes when "Alternate Forms" is off but "Costumes" is on.

### Final Variant Controls

```txt
☐ Costumes           Include costume variants e.g. Pikachu (Rock Star)
☐ Alternate Forms    Include alternate forms e.g. Arceus-Flying, Unown
☐ Female Forms       Include female variants e.g. Venusaur ♀
☐ Mega / Primal      Include temporary evolutions e.g. Mega Charizard X
```

## [4] National Dex Grouping (Client-side)

When `dex=national` (default), the grid renders with generation section headers.
This is **client-side only** — no BE change needed. FE groups entries by dex number range.

```ts
const GENERATION_RANGES = [
  { label: "Generation I",   min: 1,    max: 151  },
  { label: "Generation II",  min: 152,  max: 251  },
  { label: "Generation III", min: 252,  max: 386  },
  { label: "Generation IV",  min: 387,  max: 493  },
  { label: "Generation V",   min: 494,  max: 649  },
  { label: "Generation VI",  min: 650,  max: 721  },
  { label: "Generation VII", min: 722,  max: 809  },
  { label: "Generation VIII",min: 810,  max: 905  },
  { label: "Generation IX",  min: 906,  max: 1025 },
] as const;
```

Regional forms (Alolan Rattata = dex #19) appear under Gen I in National Dex,
grouped after their base form — consistent with how infinite scroll delivers them
(already ordered by `dexNumber ASC, formPriority ASC`).

## [5] BE Changes Required

### A. New contract field: `dex`

```ts
// packages/contexts/collection-context/src/core/definitions/pokemon-dex.ts
export type PokemonDex = (typeof POKEMON_DEXES)[number];
export const POKEMON_DEXES = [
  "NATIONAL",
  "KANTO", "JOHTO", "HOENN", "SINNOH", "UNOVA",
  "KALOS", "ALOLA", "GALAR", "HISUI", "PALDEA",
] as const;
```

```ts
// BrowsePokedexInput — add dex field:
export interface BrowsePokedexInput {
  dex?: PokemonDex; // defaults to NATIONAL
  filters?: Partial<{
    classification: PokemonClassification[];  // NEW — replaces form
    search: string;
    status: PokedexStatus;
    types: [string] | [string, string];
    variants: Partial<{
      alternateForm: boolean;       // NEW
      costume: boolean;
      gender: boolean;
      temporaryEvolution: boolean;
    }>;
  }>;
  pagination: { limit: number; page: number; };
  trainerId: string;
}
```

> `form: PokemonForm` and `regions: PokemonRegion[]` are **removed** from filters.

### B. New constants: `REGIONAL_ORIGIN_SUFFIXES`

```ts
// packages/database/src/prisma/providers/constants/regional-origin-suffixes.ts
// Used to query Hisuian/Alolan/etc forms in region-specific dexes.
export const REGIONAL_ORIGIN_SUFFIXES: Partial<Record> = {
  ALOLA:  "ALOLAN",
  GALAR:  "GALARIAN",
  HISUI:  "HISUIAN",
  KANTO:  "KANTONIAN",
  PALDEA: "PALDEAN",
};
```

### C. Adapter query logic for `dex`

```ts
// In PrismaPokedexServiceAdapter.browse():

if (dex && dex !== "NATIONAL") {
  const range = REGION_DEX_RANGES[dex];           // existing constant
  const suffix = REGIONAL_ORIGIN_SUFFIXES[dex];   // new constant

  if (dex === "HISUI") {
    // Hisui: no dex range, only regional forms
    andConditions.push({ form: { contains: "HISUIAN" } });
  } else if (suffix) {
    // Has both dex range AND regional forms (Alola, Galar, Paldea, Kanto)
    const [min, max] = range;
    andConditions.push({
      OR: [
        { dexNumber: { gte: min, lte: max }, formCategory: { not: "REGIONAL_FORM" } },
        { form: { contains: suffix } },
      ]
    });
  } else {
    // Pure dex range, no regional forms (Johto, Hoenn, Sinnoh, Unova, Kalos)
    const [min, max] = range;
    andConditions.push({ dexNumber: { gte: min, lte: max } });
  }
}
```

### D. Classification filter

```ts
const classification = filters?.classification;
if (classification && classification.length > 0) {
  andConditions.push({
    pokemonClassification: { in: classification }
  });
}
```

### E. Alternate form variant

```ts
const { alternateForm = false, costume = false, ... } = filters?.variants ?? {};

if (!alternateForm) {
  andConditions.push({
    OR: [
      { formCategory: { not: "ALTERNATE_FORM" } },
      { isCostume: true }, // costume filter is separate — don't hide costumes here
    ]
  });
}
```
